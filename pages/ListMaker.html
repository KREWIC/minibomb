Spending Plan

Objective

This would be a pretty simple master list application. I want something that's outside of applications like word or onenote namely so it's a bit more modular. You'd start at the master window where you can select from your differnet lists. When adding a new list you'll select from a number of options for the kind of list it is. Some of the options could include journal entries, date and calendar, unordered list, Key Counting, and probably some others down the line

The Code

I'd probably build this in Python using TKinter again, using a root menu that would. For the selections I'd probably use an open ended array where you can with each entry having a button to delete or edit. There's going to be a lot of data to manage here, it's probably best to make a step process for how everything is going to need to shake out. Here's a user story for how this should all work. This branch is for users creating a new list upon opening the program.

  1. The User opens the program
    • Program loads the master array
    • Likely will need a for loop that creates each selection to get this to work with TK properly.
  2. The User selects 'New List' button in the top left corner
    • Function will run to spawn the options menu
  3. The User makes selections on the type of list, potential theme(color options) and inputs a name
  4. The User selects the Create List button
    • The program will then make an addition to the master array with the title of the name.
    • Based on the selections you have made in the option menus, several things will happen
      1. A new Python file will be made and populated with a template for the list type.
      2. post creation, that file will open and run for the user to populate with list items.

The above is just one possible user story, this is a project that would require a lot of moving parts, and would likely need a roadmap with milestones for the different types of lists. A good one to do if I ever find some extra time.
